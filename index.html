About

This project is aimed at using the Network Rail Data Feeds to help learn some new technologies through an area I am interested in. I originally intended to do something with the Live Departure Board Web Service until National Rail made it pay-only. This site should not be relied on as an accurate source, although the data comes from Network Rail, I cannot guarantee the site will have interpreted the data correctly.

Web Client Technologies Used

The web client uses:

  • HTML 5 based around bootstrap
  • Theme from Bootswatch
  • jQuery
  • knockout
  • Google Maps API
  • Windows Azure Web Sites

Server Technologies Used

The server side uses:

  • Alchemy Websockets to handle websocket connections
  • Apache NMS Stomp (slighlty modified) to connect to the rail data feeds
  • ASP.NET MVC Web API for serving data to the web client
  • SQL Server 2008 R2

In Use

This site has two URLs.

  • (http://www.trainnotifier.co.uk) - this is fully hosted on a Windows Azure Virtual Machine and uses Azure SQL as a backend. This is updated when features have been tested on the site below.
  • (http://trainnotifier.azurewebsites.net) - this is hosted on the free Windows Web Sites tier. The backend runs off my own HP Microserver. This is auto-updated from github.
